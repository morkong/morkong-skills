# 在线学习资源全网搜集整理

这个技能定义了一套完整的"全网学习资源搜集、验证、整理"流程。适用于任何领域的学习或备考资源收集。

## 核心原则

在执行任何步骤之前，先理解以下设计理念：

- **质量优先于数量**：宁可每个维度只保留 5 条精品，也不堆砌 20 条低质链接。用户需要的是可用的资源，不是链接农场。
- **全量验证是底线**：每条最终输出的链接都必须经过验证。采用分层验证策略：国际站点用 WebFetch 直接验证（`✅`），国内平台基于搜索结果交叉验证（`👀`），明确失效标注 `❌`。
- **免费优先，付费标注**：优先发掘免费资源，付费资源必须标注大致价格。
- **时效性透明**：视频/文章标注发表年份，官方资源/政策标注发布时间，用户需要判断内容是否过时。
- **合法合规用资源**：对网盘/电子版资源标注版权+安全提示，对第三方平台标注免责声明，对社区经验标注仅供参考。

## 执行流程

### 阶段零：需求澄清与确认

**本阶段目标**：在投入搜索资源之前，通过分层提问深挖用户真实需求，复述对齐后再启动搜索，保证交付结果完全贴合用户预期。

#### 0.1 触发判定

统计用户原始表述中已覆盖的核心维度数量。核心维度包括：**学习目标、当前基础、资源类型偏好、特殊约束**（免费/时效性/语言）。

| 覆盖维度数 | 处理方式 |
|-----------|---------|
| ≥ 3 个 | **免提问，直接生成确认清单**。从用户表述中提取已有信息 + 未提及的维度用默认值兜底，整理为结构化确认清单请用户确认 |
| < 2 个 | **发起标准化澄清提问**。一次性抛出所有问题，用户回复选项字母即可 |

#### 0.2 标准化需求澄清话术

当用户表述不足 2 个核心维度时，发送以下提问：

```
为了给你整理最贴合的资源，我先确认几个关键点，你可以直接回复选项字母：

1. 你的核心学习目标是？
   A. 应试备考（考试/拿证/考级）
   B. 零基础入门（兴趣/了解基础知识）
   C. 进阶提升（深化技能/钻研技术）
   D. 求职就业（攒项目/准备面试）

2. 你目前的基础水平是？
   A. 完全零基础，从没接触过
   B. 略有了解，知道基本概念
   C. 有一定基础，能简单上手
   D. 已有实操经验，想系统提升

3. 你更偏好的资源形式是？（可多选）
   A. 视频教程（跟着学）
   B. 文档/书籍/笔记（系统看）
   C. 真题/题库（刷题练）
   D. 实战项目/源码（动手做）
   E. 都可以，均衡推荐

4. 有没有以下特殊要求？（可多选）
   A. 只找免费资源，排除付费内容
   B. 需要外文/英文原版资源
   C. 尽量找近1年内的最新内容
   D. 无特殊要求

有其他具体要求也可以补充~ 如果不想详细选，我也可以按通用标准直接整理。
```

**回复示例**：

```
# 用户只需回复选项字母即可，例如：

1A, 2A, 3AB, 4A

# 表示：应试备考 + 完全零基础 + 要视频和文档 + 只要免费
```

#### 0.3 需求复述与确认模板

无论走"免提问"还是"完整提问"路径，最终都必须输出确认清单请用户确认：

```
好的，我先跟你对齐下本次资源整理的需求，没问题我就启动全网搜索和链接验证：

**📌 需求确认清单**
1. 核心目标：{从用户回答提取，或默认"零基础入门"}
2. 当前基础：{从用户回答提取，或默认"略有了解"}
3. 资源偏好：{从用户回答提取，或默认"均衡推荐，优先免费资源"}
4. 特殊约束：{从用户回答提取，或默认"无，兼顾免费与优质付费，中文为主"}
5. 交付内容：结构化资源清单 + 分阶段学习计划

确认无误回复「开始」即可，有调整随时告诉我~
```

**规则**：
- 用户未明确回答的维度，使用默认值兜底，并在清单中标注「（默认配置）」
- 用户提出的个性化要求（如"只要B站视频""不要网盘资源"），需单独列入清单

#### 0.4 需求-执行参数完整映射

| 维度 | 选项 | 对应执行参数调整 |
|------|------|-----------------|
| **核心目标** | A. 应试备考 | 主题属性归为职业考证/考研学科类，强化官方资源+历年真题+题库；弱化实战项目+GitHub开源；输出完整冲刺备考方案 |
| | B. 零基础入门 | 难度设为入门级，优先视频教程+基础笔记+入门书籍；默认不搜索纯外文资源；输出零基础入门三件套 |
| | C. 进阶提升 | 难度设为进阶级，强化GitHub开源+深度博客+实战项目；加大外文资源权重；输出进阶提升路线 |
| | D. 求职就业 | 主题属性混合编程/技能类，强化实战项目+面试题库+GitHub开源；输出求职导向的项目作品集方案 |
| **基础水平** | A. 完全零基础 | 全量保留入门维度资源，剔除高难度内容；学习计划从概念科普开始 |
| | B. 略有了解 | 入门资源缩减，基础+进阶内容按 6:4 比例搭配 |
| | C. 有一定基础 | 入门内容仅做补充，核心推荐进阶+实战资源 |
| | D. 已有经验 | 跳过所有入门内容，聚焦高阶技术+源码+架构类资源 |
| **资源偏好** | 多选组合 | 选中的资源类型维度保留5条上限；未选中的维度缩减至2-3条或直接裁剪 |
| **特殊约束** | A. 仅免费 | 全维度过滤纯付费资源，仅保留免费/部分免费内容 |
| | B. 需外文资源 | 追加英文关键词搜索，优先收录外文原版+带中文翻译的优质资源 |
| | C. 近1年最新 | 质量评分中时效性权重提升至30%，优先收录近12个月内更新的内容 |

#### 0.5 兜底默认规则

若用户选择跳过提问、或部分问题未回答，按以下默认值执行：

| 维度 | 默认值 |
|------|--------|
| 核心目标 | 零基础入门（无明确说明时，优先通识入门需求） |
| 基础水平 | 略有了解 |
| 资源偏好 | 均衡推荐，优先免费资源 |
| 特殊约束 | 无，默认兼顾免费与优质付费，中文资源为主 |

#### 0.6 需求追溯

用户确认后，将确认清单中的信息记录到最终输出文档的头部元数据区域，格式如下：

```markdown
> 📋 需求匹配：核心目标={X} | 基础水平={X} | 资源偏好={X} | 特殊约束={X}
```

这确保用户清楚知道"这些资源是基于什么标准筛选的"，全链路可追溯。

### 阶段一：自适应维度搜索

**在发起搜索前，完成以下自动分析**（无需向用户提问，由用户确认清单中的信息直接推导）：

1. **阅读参考站点**：阅读 `references/known-sites.md`，了解按领域分类的优质资源站点清单。
2. **确定主题属性**，根据下表设定各维度的权重：

| 主题属性 | 特征识别 | 强化维度 | 弱化/跳过维度 | 追加搜索词 |
|---------|---------|---------|-------------|-----------|
| **职业考证类** | 软考/建造师/会计/医学/金融等 | 官方资源 ⭐⭐⭐、历年真题 ⭐⭐⭐、题库 ⭐⭐ | 实战项目 | `报名时间`、`报考条件`、`历年真题 答案解析` |
| **编程技术类** | 编程语言/框架/架构/DevOps等 | GitHub开源 ⭐⭐⭐、实战项目 ⭐⭐⭐、视频教程 ⭐⭐ | 历年真题 | `实战`、`源码`、`项目`、`练手案例` |
| **考研/学科类** | 高考/考研/期末/学科考试等 | 教材教辅 ⭐⭐⭐、笔记总结 ⭐⭐⭐、历年真题 ⭐⭐ | 实战项目 | `考研`、`期末`、`重点考点`、`历年真题` |
| **公职考试类** | 国考/省考/事业单位/选调生等 | 官方资源 ⭐⭐⭐、历年真题 ⭐⭐⭐、题库 ⭐⭐ | 实战项目、GitHub开源 | `招考公告`、`职位表`、`报名入口`、`行测`、`申论` |
| **语言学习类** | 英语/日语/韩语/法语等 | 视频教程 ⭐⭐⭐、音频学习 ⭐⭐⭐、AI辅助工具 ⭐⭐ | 历年真题、题库 | `听力`、`口语`、`词汇`、`语法`、`中文翻译版` |
| **通识/技能类** | 设计/剪辑/兴趣等（以上之外） | 视频教程 ⭐⭐⭐、社区经验 ⭐⭐ | 历年真题、题库 | 无特殊追加 |

3. **判断外文资源需求**：编程技术/前沿学术/国际认证类主题自动加大英文搜索权重，追加 `{主题英文} tutorial free course {当年} {前一年}` 等关键词，并搜索中文翻译版。

4. **应用阶段零的参数映射**：根据 0.4 节的需求-执行参数映射表，将用户的每个选项转化为具体的搜索策略调整。

然后**同时**从以下维度发起 WebSearch。强化维度使用更多搜索词变体，弱化维度可减少搜索词。

| # | 维度 | 搜索词模板 | 目标内容 | 最低保留标准 |
|---|------|-----------|---------|-------------|
| 1 | 视频教程 | `{主题} 免费视频教程 B站 {当年} {前一年}` | B站/慕课网/YouTube 视频课程 | 至少 3 条，来自真实课程平台 |

| 8 | GitHub开源 | `{主题} site:github.com 笔记 合集 资源` | 开源笔记仓库、刷题代码 | 优选 star 数高、持续更新的仓库 |
| 9 | 实战项目 | `{主题} 实战项目 练手案例 源码` | 项目实战教程、练手项目、源码仓库 | 代码可运行、有文档说明、难度分层 |
| 10 | 网盘资源 | `{主题} PDF 电子版 百度网盘 阿里云盘` | 电子书、讲义、课件分享 | 注意标注提取码是否有效 |
| 11 | 音频学习 | `{主题} 播客 音频课程 喜马拉雅` | 音频课程、播客节目 | 仅收录内容相关性强的音频资源 |
| 12 | AI辅助工具 | `{主题} AI备考 AI刷题 GPT 智能学习` | AI 驱动的学习/刷题工具 | 排除纯概念营销，只保留可实际使用的工具 |

**强化/弱化规则**：
- **强化维度**（⭐⭐⭐）：使用 2-3 个搜索词变体
- **普通维度**（⭐⭐）：使用默认搜索词即可
- **弱化/跳过维度**：使用简化搜索词

**重要**：以上搜索应**并行发起**。不要一个接一个地搜索。

### 阶段二：补充收网（1-2 轮）

初轮搜索完成后，根据发现的优质线索进行定向深挖：

1. **UP主/作者深挖**：如果某个 UP 主或作者贡献了多条优质内容，用 `site:{平台域名} {作者名} {主题}` 深挖其全部相关作品。
2. **平台深挖**：对发现的每个刷题/课程平台，单独搜索确认其免费/付费策略和用户口碑。
3. **长尾补充**：如果某个强化维度的优质结果不足 3 条，调整搜索词重新搜索该维度。
4. **官方信息核对**：对考试类主题，搜索官方发布的报名时间、报考条件、考试大纲等最新信息，并记录发布时间。
5. **外文资源翻译版搜索**：如果阶段零判断需加大外文权重，对找到的外文优质资源搜索其中文翻译版。

### 阶段三：三级质量过滤

必须严格执行，不能跳过任何一级。

#### 第一级：相关性过滤

检查每条搜索结果的标题和摘要：
- ✅ 标题或摘要明确包含目标主题关键词 → 通过
- ✅ 内容明显与学习相关 → 通过
- ❌ 纯广告/营销页面（标题含"保过""包过""秒杀"等营销词，且无实质内容）→ 丢弃
- ❌ 标题与主题有关但摘要显示完全不相关 → 丢弃
- ❌ 付费课程平台的纯销售落地页 → 标注为"付费"，保留但降低推荐值

#### 第二级：链接可访问性验证（分层验证）

**注意**：WebFetch 工具对国内网站（B站、微信公众号、知乎、百度网盘、夸克网盘等）几乎全部返回错误，这不是链接失效，而是工具限制。因此采用**分层验证策略**：

**A. 可验证站点**（使用 WebFetch 实际访问）：
- 国际站点：GitHub、GitLab、Stack Overflow、Medium、Dev.to 等
- 国外大学/机构：.edu 域名、MIT OCW、Coursera、edX 等
- 技术文档站：MDN、freeCodeCamp、GeeksforGeeks 等
- 搜索引擎返回的普通 HTTP 网站

**B. 国内平台**（基于搜索结果摘要交叉验证，标注 `👀 待确认`）：
- B站 / 微信公众号 / 知乎 / 豆瓣 — WebFetch 几乎必失败
- 百度网盘 / 夸克网盘 / 阿里云盘 — 需要登录，WebFetch 不可用
- 微信小程序 / App Store 链接 — WebFetch 不可用

对 B 类资源，验证方式为：
1. 在 WebSearch 结果中至少被 2 个不同来源交叉提及
2. 标题和描述信息完整（非空白摘要）
3. 标注 `👀 待确认`（而非"验证失败"），表示"搜索结果确认存在，但受限于工具无法直接访问页面"

**C. 明确失效**（标注 `❌ 链接失效`）：
- WebFetch 返回 404 / 域名不存在 / 连接超时（国际站点）
- 搜索结果摘要明确标注"已删除""已下架""链接失效"
- URL 格式明显错误（不完整的链接）

**验证状态三态说明**：
- `✅ 已验证` — WebFetch 成功访问并确认内容匹配
- `👀 待确认` — 国内平台/需登录，基于搜索结果交叉验证确认存在，建议用户自行点击确认
- `❌ 链接失效` — 确认无法访问或内容已删除

#### 第三级：质量评分和排序

对验证通过的链接按以下标准排序：

- **内容丰富度（40%）**：视频时长/文章字数/题目数量/仓库文件数
- **来源权威性（30%）**：官方来源 > 知名平台 > 个人博客 > 匿名分享
- **用户口碑（20%）**：点赞/收藏/star 数、评论区好评率
- **时效性（10%）**：当前年份 > 去年 > 更早

每个维度**最多保留 5 条**最优结果。不足 3 条时如实保留，不要凑数。

### 阶段四：结构化整理

将过滤后的资源按类别整理为 Markdown 表格。

#### 通用表格字段（所有维度）

| 字段 | 说明 |
|------|------|
| **标题** | 资源名称，格式为 `[资源名称](URL)`，名称要精炼且能识别来源平台。**优先使用可点击链接**。若因平台限制（如微信公众号文章需关注后查看、App内部页面、需要登录的网盘等）确实无法获取直接URL，则在标题后附搜索指引，如 `公众号搜索"xxx"`、`App内搜索"xxx"`，确保用户能找到该资源 |
| **来源** | 资源所在平台或网站，如 `B站`、`知乎`、`微信公众号`、`百度网盘`、`夸克网盘`、`GitHub`、`喜马拉雅`、`中国大学MOOC` 等，让用户一眼知道去哪里找 |
| **描述** | 1-2 句话说明内容和特色亮点，包含视频时长/文章字数等具体信息 |
| **适用阶段** | 入门 / 进阶 / 冲刺，根据资源内容难度标注，方便用户按自身进度匹配 |
| **推荐值** | ⭐⭐⭐⭐⭐（五星制），免费且内容全面者给五星 |
| **是否付费** | 免费 / 付费（约 ¥XX）/ 部分免费 |
| **验证状态** | ✅ 已验证 / ❌ 链接失效 / 👀 待确认（详见阶段三验证说明） |

#### 官方资源专用字段

官方资源、政策类、考试大纲类在通用字段基础上增加：

| 额外字段 | 说明 |
|---------|------|
| **发布时间** | 该政策/大纲/通知的具体发布时间，如 `2025-03`，让用户直接判断时效性 |

**官方资源的"描述"字段直接填写该政策/大纲的核心要点摘要，而不是泛泛介绍。**

#### 空维度处理

如果某维度确实没有找到符合条件的资源，**不要跳过**，而是保留标题并附说明：

```markdown
## 十一、音频学习资源（0 条）

> ℹ️ 本主题未检索到符合质量标准的相关音频学习资源，暂不收录。
```

这能让用户明确知道"搜过了但没有好结果"，而非"漏搜了"。

**注意**：空维度说明及其他 `> ℹ️` 提示是面向最终用户输出的内容，**不要泄露内部决策逻辑**（如"本维度不适用，已跳过""用XX替换"等）。面向用户的说明应客观描述资源情况即可。

### 阶段五：输出合规与风险提示

合规提示分为两类：**维度级**（放在对应表格下方）和**内容级**（放在每条资源的描述中）。

#### 维度级提示（放置在对应资源表格正下方）

**网盘资源**（维度十）表格下方：
> ⚠️ **版权提示**：本表格中的网盘资源及电子版教材为网络公开分享，仅作个人学习参考，禁止商用，请支持正版。
> ⚠️ **安全提醒**：网盘资源可能存在捆绑软件、恶意文件风险，下载后请自行核验文件安全性，建议先用杀毒软件扫描后再打开。

**AI 工具 / 非官方题库**（维度四、十二）表格下方：
> ⚠️ **第三方平台提示**：本表格中的 AI 工具及非官方题库为第三方平台，使用前请自行评估数据安全与隐私风险。
> ⚠️ **准确性提醒**：AI 生成的答案与解析可能存在疏漏，仅供参考，核心知识点请以官方教材为准。

**官方资源 / 报考政策**（维度六）表格下方：
> ⚠️ **时效提醒**：以上报考政策、报名条件及考试大纲等信息具有时效性，请以官方最新通知为准。表中"发布时间"字段可供判断信息新旧。

**社区经验**（维度七）表格下方：
> ⚠️ **经验参考提醒**：以上内容为个人经验分享，仅供参考，请结合自身情况调整，不构成通用备考建议。

### 阶段六：输出分层学习建议

在所有资源表格之后，附加 **学习建议总结**。根据阶段零确定的难度分级调整内容侧重：

#### 核心三件套（按起点拆分）

**在列出具体资源之前，先用一个简要表格解释筛选维度**，让用户知道这些推荐是基于什么标准挑出来的，而非主观偏好：

```markdown
> 💡 **推荐维度**：以下三件套资源综合以下 6 个维度筛选：
>
> | 维度 | 说明 |
> |------|------|
> | 难度匹配 | 是否贴合用户基础水平 |
> | 形式互补 | 视频/教材/题库是否覆盖输入—理解—输出的完整闭环 |
> | 付费门槛 | 免费优先，付费标注 |
> | 体系完整度 | 覆盖的知识模块是否全面 |
> | 用户口碑 | 播放量/收藏/社区推荐热度 |
> | 版本时效 | 发布时间是否适配最新教材和考纲 |
```

然后给出三件套内容，并在每个阶段解释**为什么从同维度的多条资源中选了这一条而没选其他的**（如：从视频教程5条中挑了A而非B，是因为A真正做到了零基础可跟而B节奏偏快）。

```markdown
### 🎯 零基础入门三件套
（面向入门级用户，精选 3 条最容易上手的免费资源）
1. ...
2. ...
3. ...

### 🏆 冲刺备考三件套
（面向应试级用户，精选 3 条最高效的真题/笔记/题库资源）
1. ...
2. ...
3. ...
```

如果用户主题不涉及应试，可省略"冲刺备考三件套"。如果是纯应试主题，两项都保留——入门三件套帮打基础，冲刺三件套帮提分。

#### 分阶段计划 + 资源搭配逻辑

每个阶段不仅推荐资源，还要**解释为什么要这样搭配**：

```markdown
### 🗺️ 分阶段学习计划

#### 阶段一：基础入门（建议 X 周）
- 推荐资源：...
- **资源搭配逻辑**：先用 [视频A] 搭建知识框架，建立整体认知；再用 [教材B] 补全细节和公式推导；学完每章后用 [题库C] 做 5-10 道基础题巩固概念。视频 → 教材 → 题库三者形成"理解→深入→验证"闭环。

#### 阶段二：强化提升（建议 X 周）
- 推荐资源：...
- **资源搭配逻辑**：...（解释）

#### 阶段三：冲刺实战（建议 X 周）
- 推荐资源：...
- **资源搭配逻辑**：...（解释）
```

"资源搭配逻辑"的核心是解释「资源 A 负责什么 → 资源 B 弥补 A 的什么不足 → 资源 C 如何检验 A+B 的效果」，让用户理解为什么要同时用这几份资源，而非堆链接。

## 注意事项

1. **搜索词必须与主题相关**：每次使用时根据用户指定的主题替换 `{主题}`
2. **年份动态替换**：搜索词模板中的 `{当年}` 替换为当前年份，`{前一年}` 替换为上一年份（例如当前是 2026 年，则使用 2026 和 2025）。这确保搜索到的资源具有时效性。
3. **根据主题属性+难度分级调整权重**：职业考证类强化真题和官方资源，编程技术类强化 GitHub 和实战项目，入门级优先视频教程，应试级优先真题题库
3. **外文资源不遗漏**：编程/技术/国际认证类主题必须搜索英文关键词并尝试找中文翻译版
4. **去重合并**：同一资源出现在多个维度时，只保留在相关性最高的维度下
5. **空维度不跳过**：保留标题 + "本主题未检索到符合质量标准的相关资源，暂不收录"
6. **表格字段完整**：通用维度 7 字段（标题/来源/描述/适用阶段/推荐值/是否付费/验证状态），官方资源 8 字段（+发布时间）。不可遗漏任何字段
7. **链接格式**：优先使用 Markdown 内联链接格式 `[标题](URL)`。若因平台限制（需登录、App内部页面、关注后查看等）无法获取直接URL，用搜索指引代替，如 `公众号搜索"xxx"`、`App搜索"xxx"`、`网盘搜索"xxx"`。不可因拿不到URL就直接放弃该资源
8. **来源列必填**：每条资源必须标注来源平台，如 `B站`、`知乎`、`微信公众号`、`百度网盘`、`夸克网盘`、`GitHub`、`喜马拉雅`、`中国大学MOOC` 等，不得留空
8. **合规标注不可遗漏**：网盘资源（版权+安全）、AI工具/非官方题库（第三方+准确性）、社区经验（个人经验参考）、官方政策（时效提醒）四类标注必须添加
9. **资源搭配逻辑必须写**：分阶段计划中每个阶段都要解释为什么这样搭配资源，不能只罗列链接
10. **文件开头必须有 [TOC]**：12 个维度篇幅较长，目录帮助用户快速跳转
